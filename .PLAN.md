# Graph exploitation ideas

## Tier 1 — Highest signal, directly actionable

1. **NetworkX + Gephi/PyVis export** — Load the pickle into NetworkX, export GEXF or GraphML. Gephi modularity clustering immediately reveals topic communities across your conversation history. One-shot insight into how your intellectual domains cluster.

2. **Entity timeline / drift analysis** — Each message has conversation_id → timestamp (recoverable from the raw JSON). Plot entity frequency over time. Answers: what topics have I been obsessing over? When did "lattice" stop dominating and "eBPF" take over?

3. **Central entities by betweenness centrality** — Which entities bridge otherwise-disconnected clusters? High betweenness = cross-domain concepts. Expect things like "Python", "hash", maybe "n" (math/code overlap) — but also surprises.

4. **Conversation similarity via shared entity/keyword fingerprints** — Jaccard or cosine on entity/keyword sets per conversation → conversation embedding. Find "conversations I had twice without realizing it." Useful before starting a new topic.

## Tier 2 — Medium effort, high payoff

5. **Neo4j / Kuzu ingest** — The graph dict maps directly to a property graph. Cypher queries then replace ad-hoc Python. `MATCH (e:Entity)<-[:MENTIONS]-(m:Message)<-[:CONTAINS]-(c:Conversation) WHERE e.name = 'fpylll'` — instant provenance for any topic.

6. **Keyword co-occurrence → topic model** — The `edges_cooc` set is already a co-occurrence graph over entities. Run community detection (Louvain) on it → emergent topic clusters without LDA. Compare to TF-IDF keywords to validate.

7. **Reply-chain depth / conversation structure metrics** — `edges_replies_to` is a DAG. Compute: longest chain, branching factor, depth distribution. Identifies conversations where you explored deeply vs. breadth-first.

8. **Cross-provider join + graph diff** — Use `join` to merge your Claude + ChatGPT exports, then build two separate graphs and diff entity/keyword sets. What do you discuss with Claude vs. GPT? Are there systematic topic separations?

## Tier 3 — Heavier, research-grade

9. **Graph neural embedding (node2vec / GraphSAGE)** — Embed message nodes → dense vectors → ANN search. "Find me messages semantically similar to this one" as a retrieval primitive. Basically what mnemo-mcp does but over the graph structure rather than raw text.

10. **Temporal knowledge graph** — Add timestamps as node/edge attributes, run TGNN or simple sliding-window analysis. Detect when concepts "activate" together, how long topic bursts last, whether there are recurring seasonal patterns in your research.

11. **LLM-over-graph** — Run the graph through a Cypher-aware LLM agent. Natural language queries over your full conversation history: "What did I conclude about HNSW early termination?" with entity-linked provenance.

Recommended first move: add a `convo_tools/_export.py` with NetworkX ingest + GEXF writer, then feed into Gephi for the first visual. Takes ~2 hours and immediately makes the graph legible.
